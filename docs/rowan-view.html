REPOSITORIES — the disk lens

Repos you track, added once via ＋ (clone a git URL, or point at a folder). Tracking ≠ loading: the row shows its load spec, and hovering a git clone reveals ↻ Update from Git and ▶ Load into Image. A repo already in the connected stone says · loaded; broken states say so plainly.

↻ UPDATE FROM GIT

A repo Jasper cloned from a URL can be refreshed in place — a fast-forward pull that also updates submodules (a Rowan project may vendor its packages that way). Handy when the project gains a file your clone predates. Locally-checked-out folders have no remote, so they get no update button.

⚠ WHEN A LOAD WON'T FIT

Loading a big project builds its whole transient object graph before the final commit, and overflows a small gem's temp-object cache (“VM temporary object memory is full”). A project can declare the cache it needs in rowan/gemstone.ston (#minTempObjCacheKB — a GemStone-platform STON file Rowan ignores). When the connected gem falls short the repo gets a warning triangle, and pressing Load first opens a dialog with the fix rather than grinding through a doomed load. Databases Jasper creates get a 500 MB gem cache, and Jasper raises an existing database's cache to 500 MB when it starts the stone — so a Jasper-managed stone meets the declaration and never trips the warning.

LOADED PROJECTS — the image lens

What the connected stone actually has. A dirty project reads exactly like a modified file in the git view — tinted name plus a right-aligned M, in the real gitDecoration theme colors. The image's own projects fold into a collapsed Built-in group and can't be unloaded. Right-click a project of yours (shown open) for Diff, Export Copy…, and Unload….

CHANGES — the bridge

Image↔disk drift for your projects (built-ins are read-only image trees, so they stay out), in git vocabulary: A only in the image, D only on disk, M differs — tinted rows, class and method icons. The section starts collapsed; a project's diff runs when you expand it, cached until the next refresh. This is where commit-to-disk staging will live.

ONE QUERY PER REFRESH

Repositories and Loaded Projects share a single image query per refresh cycle; diffs run once per project per cycle. Nothing polls.
